## Comment Response Templates

### "How does semantic search work?"

```
Great question! It uses Apple's NLEmbedding framework to generate 50-dimensional semantic vectors for each clipboard item.

When you search, it compares the search query's embedding to stored items using cosine similarity. Similar concepts end up close together in the vector space, even if they use different words.

Example: "coffee" and "espresso" have similar vectors, but "coffee" and "bicycle" don't.

All happens locally in <50ms. No API calls needed!

More technical details in the README: [link to architecture section]
```

### "Is it really 100% private?"

```
Yes! You can verify this yourself since the code is open source.

Here's what to check:
1. Search the codebase for URLSession, Alamofire, network libraries ‚Üí you won't find any
2. Run Little Snitch or a network monitor ‚Üí zero connections
3. Review the ML code ‚Üí all uses Apple's NLEmbedding (local)

The only "AI" is Apple's on-device frameworks. No OpenAI, no Anthropic, no cloud.

Your clipboard data is as private as any other file on your Mac.
```

### "Will this work with [other app]?"

```
Yes! ClipboardManager works system-wide with any Mac app.

It monitors NSPasteboard (the system clipboard), so it captures everything you copy from any application.

You can also exclude specific apps in Settings if needed (e.g., password managers).
```

## FAQ Preparation

Have answers ready for common questions:

**Q: macOS version required?**
A: macOS 13.0 (Ventura) or later

**Q: Does it work with [specific app]?**
A: Yes, works system-wide with all Mac apps

**Q: Can I sync across devices?**
A: Not yet - it's local-only for privacy. iCloud sync with E2E encryption is on the roadmap

**Q: How do I uninstall?**
A: Drag from Applications to Trash. Data is in ~/Library/Application Support/ClipboardManager if you want to delete that too

**Q: Is there an iOS version?**
A: Not yet, focusing on perfecting the Mac experience first

**Q: Can I contribute?**
A: Yes! It's open source. PRs welcome: [GitHub link]

**Q: How does this compare to Alfred's clipboard?**
A: ClipboardManager has semantic search and context detection, which Alfred lacks. Alfred is more of a launcher with clipboard features

**Q: What about Raycast?**
A: Similar - Raycast is a launcher with clipboard. ClipboardManager is purpose-built for clipboard with AI features
